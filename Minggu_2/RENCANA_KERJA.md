# 🗓️ Detail Rencana Kerja: Minggu 2

## Tema: Inisialisasi & Arsitektur Teknis

Minggu ini berfokus pada persiapan "Skeleton" proyek, baik dari sisi bisnis maupun teknis, untuk memastikan pondasi aplikasi kuat sebelum masuk ke tahap desain interface.

---

### 👤 Anggota 1: Business Strategist & UI/UX Designer

**Fokus: Analisis Strategis & Alur Pengguna**

1. **Analisis SWOT**: Membuat dokumen mendalam mengenai _Strengths_ (Kekuatan), _Weaknesses_ (Kelemahan), _Opportunities_ (Peluang), dan _Threats_ (Ancaman) untuk platform e-commerce outdoor ini.
2. **User Journey Diagram**: Memetakan langkah-langkah pengguna mulai dari landing page, mencari produk, hingga proses transaksi selesai.
3. **Dokumentasi Bisnis**: Menyimpan hasil analisis ke dalam folder `Minggu_2/Bisnis/`.

### 👤 Anggota 2: Frontend Developer & Animation Expert

**Fokus: Struktur Basis & Standarisasi Kode**

1. **Refinement Next.js & Tailwind**: Memastikan konfigurasi Tailwind (colors, fonts) sudah sesuai dengan tema _Earthy Premium_.
2. **Implementasi Page-Centric Structure**: Menata folder `app/` agar setiap halaman memiliki komponen dan gaya mandiri sesuai aturan di `project_plan.md`.
3. **Setup Framer Motion**: Melakukan instalasi dan pengetesan dasar animasi untuk memastikan transisi antar halaman berjalan mulus.

### 👤 Anggota 3: Backend Developer & Database Engineer

**Fokus: Infrastruktur Data & Keamanan**

1. **Setup Supabase Project**: Membuat project baru di dashboard Supabase, mengonfigurasi API Keys, dan menghubungkannya dengan file `.env.local`.
2. **Perancangan ERD (Entity Relationship Diagram)**: Membuat skema database awal (Tabel: Users, Products, Categories, Orders).
3. **Konfigurasi Auth**: Menyiapkan penyedia autentikasi (Email/Password) di Supabase dan merencanakan Middleware untuk proteksi halaman admin/dashboard.

---

> [!IMPORTANT]
> Seluruh anggota tim diharapkan melakukan **Sync Up** setiap 2 hari sekali untuk memastikan integrasi antara desain (Anggota 1), struktur (Anggota 2), dan database (Anggota 3) berjalan selaras.

> [!TIP]
> Gunakan folder `Minggu_2/` di repository ini untuk mengunggah file-file dokumentasi (PDF, Image, atau Markdown) agar bisa dipantau oleh seluruh anggota tim.
